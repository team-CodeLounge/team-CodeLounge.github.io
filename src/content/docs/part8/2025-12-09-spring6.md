## @RestController
- `@Controller` + `@ResponseBody`를 합친 어노테이션이다.  
- 리턴값을 View로 렌더링하지 않고 **HTTP Response Body에 그대로 직렬화**해 내려준다.  
- `String`, `Map`, DTO 객체 등을 반환하면 **Jackson이 자동으로 JSON으로 변환**하여 응답한다.

## @RequestMapping("/api/v1/auth")
- 이 컨트롤러에 속한 모든 메서드의 공통 URL prefix를 정해주는 역할
- 이 클래스 안의 모든 핸들러 메서드는 "/api/v1/auth/..." 로 시작하는 URL을 가지게 된다

## @GetMapping("/test")
- HTTP 메서드 **GET** 요청을 처리한다
- 실제 요청 URL은 `/api/v1/auth/test`가 된다

## @Getter/@Setter(lombok)
- Lombok이 자동으로 getter/setter 메서드를 생성해주는 기능
- 코드에 메서드를 일이이 작성하지 않아도 된다

## @Configuration
- 이 클래스를 스프링 설정 클래스로(Config Class)등록한다는 의미
- 스프링이 빈(Bean)으로 등록해서 프로젝트 전반에서 사용할 수 있게 만든다

## ConfigurationProperties(prefix = "google")
- `application.yml`에 있는 google: 아래의 설정값들을 자동으로 이 클래스에 매핑해주는 어노테이션

## @Service
- 해당 클래스를 스프링의 서비스 레이어 컴포넌트로 등록
- 스프링이 자동으로 **빈(Bean)** 으로 생성하여 다른 곳에서 DI로 주입받을 수 있게 한다.

## @RequiredArgsConstructor
- final 필드들을 대상으로 생성자를 자동으로 생성해주는 어노테이션
- 생성자 주입 방식을 깔끔하게 유지할 수 있으며  
  `private final GoogleProperties googleProperties` 같은 필드가 자동으로 DI 된다.

## @PostConstruct
- 스프링이 이 빈을 모두 초기화한 직후 단 한번 호출되는 메서드

## Bean이란
- **스프링 컨테이너가 생성하고 관리하는 객체**를 의미한다.  
- 개발자가 직접 `new`로 만들지 않아도 되고, 컨테이너가 **생성·초기화·소멸까지 전 과정을 관리**한다.  
- 주로 `@Component`, `@Service`, `@Repository`, `@Controller`, `@Configuration` 등의 어노테이션을 통해 자동으로 빈으로 등록된다.  
- 빈은 기본적으로 **싱글톤**으로 관리되어 애플리케이션 전체에서 재사용된다.

## @JsonProperty("picture") (Jackson)
```java
@JsonProperty("picture")
private String pictureUrl;
```
- JSON에서 넘어오는 필드 이름과 자바 필드 이름이 다를 때 매핑을 연결해주는 어노테이션.
- JSON의 "picture" → Java의 pictureUrl 로 매핑됨

## Builder
```java
new GoogleIdTokenVerifier.Builder(...)
```
- 복잡한 객체를 단계별 설정을 적용하여 생성할 수 있게 해주는 디자인 패턴
- 마지막에 `build()`를 호출하면 완성된 객체를 반환
- 생성자에 파라미터가 너무 많아지면 가독성이 떨어지는데 이떄 Builder로 해결
- 설정을 명확하게 표현 가능

## JacksonFactory
- JSON 파싱하는 Jackson을 위한 Factory.
- 구글 API 클라이언트에서 JSON 처리를 위해 필요.

## verify(idToken)
- idToken 문자열을 실제 GoogleIdToken 객체로 변환하면서 검증까지 수행
- 검증 실패 → null 반환

## @EntityListeners(AuditingEntityListener.class)
- JPA 엔티티의 생성 시간(createdAt), 수정 시간(updatedAt) 을 자동으로 기록하도록 도와주는 리스너를 등록한다
- 즉, 엔티티가 DB에 저장되거나 업데이트될 때 스프링이 자동으로 날짜를 넣어준다

## @MappedSuperclass
- 이 클래스를 단독으로 테이블로 만들지 않고, 상속받는 엔티티들이 createdAt, updatedAt 필드를 가져가도록 하는 JPA 어노테이션.

## @CreatedDate
- 엔티티가 처음 저장될 때(created) 시간을 자동으로 기록해주는 어노테이션

## @LastModifiedDate
- 엔티티가 업데이트될 때마다 시간을 자동으로 갱신해주는 어노테이션

## @Column(name = "created_at", nullable = false, updatable = false)
- created_at은 처음 생성된 이후 변하지 않는 값.

## @Column(name = "updated_at")
- 수정 시간(updated_at) 컬럼 이름 지정
- 기본적으로 null 허용(최초 생성 시 수정 시간이 없을 수 있음)

## @Temporal(TemporalType.TIMESTAMP)
- Date / Calendar 타입의 날짜를 DB 타입(TIMESTAMP 등)으로 매핑할 때 필요했던 어노테이션
