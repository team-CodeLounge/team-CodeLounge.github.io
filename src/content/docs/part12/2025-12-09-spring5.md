## 기본 흐름
```bash
Client
   ↓ JSON Request
Request DTO
   ↓
Controller
   - HTTP 요청 처리
   - Validation(@Valid)
   - DTO → Service로 전달
   ↓
Service
   - 비즈니스 로직
   - 트랜잭션 처리
   - Entity 생성/변환
   ↓
Entity
   ↓
Repository
   - JPA로 DB 접근
   ↓
Database
   ↓
Entity
   ↓
Response DTO
   - Entity → ResponseDTO 변환
   ↓ JSON Response
Client
```

## RequestDTO
- 클라이언트가 보내는 데이터 구조
- Controller에서 @RequestBody로 받음

## Controller
- HTTP요청 입구
- RequestDTO 검증(@Valid)
- Service 호출

## Entity
- DB 테이블과 매핑되는 도메인 객체

## Repository
- JPA 인터페이스
- DB CRUD 수행
- 비즈니스 로직은 포함하지 않음

## Response DTO 
- Service -> Controller -> Client로 전달되는 응답 데이터 구조
- Entity를 그대로 노출하지 않기 위해 사용
