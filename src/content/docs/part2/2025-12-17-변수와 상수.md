## 변수(var)
- 값이 변경 가능합니다(mutable).
- 선언 후 다른 값으로 재할당할 수 있습니다.
```swift {lineNumbers=false}
var age = 26
age = 27
```

## 상수(let)
- 값이 불변합니다(immutable).
- 초기화 이후에는 동일한 식별자에 다른 값을 할당할 수 없습니다.
```swift {lineNumbers=false}
let name = "donghyeon"
name = "kim"
```  
<br/>

## 값타입(struct, enum)
- 값 타입은 전체 인스턴스가 하나의 값으로 간주됩니다.  
- 따라서 let으로 바인딩하면 내부 프로퍼티가 var이더라도 변경 할 수 없습니다.
```swift
let user = User(name: "홍길동")
user.name = "홍길동2"   // ❌ 오류
```
<br/>

## 참조타입(class)
- let은 참조 자체만 불변입니다.  
- 참조가 가리키는 객체는 별개의 메모리 영역이므로 객체의 내부 프로퍼티는 변경할 수 있습니다.
```swift
class Person {
    var age = 20
}

let people = Person()
people.age = 30
people = Person()   // ❌ 오류
```
<br/>

## let은 컴파일러에게 최적화 힌트를 제공
컴파일 타임에 결정되는 let
- Swift 컴파일러는 `초기값이 컴파일 타임에 확정되는` let을 정적 분석을 통해 최적화할 수 있습니다.   
(상수 전파, constant folding)
```swift
let a = 10
let b = "hello"
let c = true
let d = 3.14
let e = [1, 2, 3]
let x = 1 + 2 + 3 + 4 
```
<br/>

컴파일 타임에 결정되지 않는 let
- 초기값이 런타입에서만 알 수 있는 값이면 `let`이라도 컴파일 타임 최적화가 적용되지 않습니다.
```swift
func getNumber() -> Int { Int.random(in: 1...100) }
let value = getNumber()  // ❌ 컴파일 시간엔 어떤 값인지 모름
```
